Names <del>Don't Matter</del> Totally Matter

Dec 14, 2017 1 min read

Now more than ever, I try to add an element of programming in most courses, which ranges from simple function calls in R to using dplyr as part of simulation. At some point in each of these classes, I find myself making two points about variable names.

Point 1: Names don’t matter.

It is important for students to know when they are required to use specific names (built-in function, functions from a library) and when they have the power to pick their own names. When programming in Python, I have developed a set of naming habits. For example, if I want to compute the monthly pay for a list of salaries, I would probably write the following.

monthly_pay  = [salary/12 for salary in salaries]

Similarly, when processing a table, I am likely to use for row in table in a loop or comprehension. I have found that some students will begin to think that they need to use the name row when working with a table. We need to make sure students understand when they hold the power to select a name. It is entirely possible to perform the above computation as follows,

unicorn  = [bob/12 for bob in snowflakes]

but that brings us to the second point.

Point 2: Names totally matter.
